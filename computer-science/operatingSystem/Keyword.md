## 1. 운영체제 개요

| 키워드                      | 설명                                                                                      |
| --------------------------- | ----------------------------------------------------------------------------------------- |
| **운영체제 (OS)**           | 하드웨어를 관리하고 응용 프로그램과 사용자 사이의 인터페이스를 제공하는 시스템 소프트웨어 |
| **커널 (Kernel)**           | 운영체제의 핵심 — 프로세스 / 메모리 / 저장장치 등 자원을 직접 관리하는 부분               |
| **이중 모드**               | 사용자 모드(User Mode) / 커널 모드(Kernel Mode) — 자원 보호를 위한 CPU 실행 모드 구분     |
| **시스템 콜 (System Call)** | 사용자 프로그램이 커널 기능(파일 읽기, 프로세스 생성 등)을 요청하기 위한 인터페이스       |
| **인터럽트 (Interrupt)**    | 하드웨어/소프트웨어 이벤트가 발생했을 때 CPU에 알려 처리를 요청하는 신호                  |
| **모놀리식 커널**           | 모든 OS 기능이 하나의 커널 공간에서 동작 — 성능 우수, 유지보수 어려움 (Linux)             |
| **마이크로커널**            | 최소 기능만 커널에 두고 나머지는 사용자 공간에서 동작 — 안정성 우수, 성능 저하            |
| **운영체제 유형**           | 일괄처리 / 시분할 / 실시간 / 분산 운영체제                                                |

## 2. 프로세스

| 키워드                                  | 설명                                                                                                  |
| --------------------------------------- | ----------------------------------------------------------------------------------------------------- |
| **프로세스**                            | 실행 중인 프로그램 — 코드 / 데이터 / 힙 / 스택 영역으로 구성                                          |
| **프로그램 vs 프로세스**                | 프로그램은 디스크에 저장된 정적 파일 / 프로세스는 메모리에 올라가 실행 중인 동적 상태                 |
| **PCB (Process Control Block)**         | 프로세스 상태 / PID / PC / 레지스터 값 / 메모리 정보 등 프로세스 정보를 담은 자료구조                 |
| **프로세스 상태**                       | 생성(New) → 준비(Ready) → 실행(Running) → 대기(Waiting) → 종료(Terminated)                            |
| **컨텍스트 스위칭 (Context Switching)** | CPU를 다른 프로세스에 할당하기 위해 현재 프로세스 상태를 PCB에 저장하고 다음 프로세스를 복원하는 과정 |
| **fork() / exec()**                     | 프로세스 생성 시스템 콜 — fork()는 부모를 복제, exec()는 새 프로그램으로 교체                         |
| **IPC (Inter-Process Communication)**   | 프로세스 간 통신 방법 — 파이프 / 메시지 큐 / 공유 메모리 / 소켓                                       |
| **좀비 프로세스**                       | 실행이 끝났지만 부모 프로세스가 종료 상태를 수거하지 않아 PCB가 남아있는 프로세스                     |
| **고아 프로세스**                       | 부모 프로세스가 먼저 종료된 자식 프로세스 — init 프로세스가 입양                                      |
| **프로세스 주소 공간**                  | 코드(Code) / 데이터(Data) / 힙(Heap) / 스택(Stack) 영역                                               |

## 3. 스레드

| 키워드                         | 설명                                                                                                       |
| ------------------------------ | ---------------------------------------------------------------------------------------------------------- |
| **스레드 (Thread)**            | 프로세스 내의 실행 단위 — 프로세스의 자원을 공유하며 독립적으로 실행                                       |
| **멀티스레드**                 | 하나의 프로세스 안에서 여러 스레드가 동시에 실행되는 방식                                                  |
| **스레드 구성**                | 스레드마다 독립적인 스택 / 레지스터 / PC를 가짐. 코드 / 데이터 / 힙은 공유                                 |
| **멀티프로세스 vs 멀티스레드** | 멀티프로세스는 독립 메모리 사용으로 안전하나 오버헤드 큼 / 멀티스레드는 자원 공유로 효율적이나 동기화 필요 |
| **사용자 수준 스레드**         | 커널 개입 없이 라이브러리가 관리 — 빠르지만 하나가 블록되면 전체 블록                                      |
| **커널 수준 스레드**           | 커널이 직접 관리 — 하나가 블록돼도 다른 스레드 실행 가능                                                   |
| **스레드 안전 (Thread-safe)**  | 여러 스레드가 동시에 접근해도 결과가 항상 올바른 코드/함수                                                 |

## 4. CPU 스케줄링

| 키워드                         | 설명                                                                                                 |
| ------------------------------ | ---------------------------------------------------------------------------------------------------- |
| **CPU 스케줄링**               | 준비 큐의 프로세스 중 다음에 CPU를 할당할 프로세스를 결정하는 것                                     |
| **선점형 vs 비선점형**         | 선점형: 실행 중인 프로세스를 강제로 중단 가능 / 비선점형: 프로세스가 스스로 CPU를 반납할 때까지 대기 |
| **FCFS**                       | First Come First Served — 먼저 온 순서대로 처리. 호위 효과(Convoy Effect) 발생 가능                  |
| **SJF**                        | Shortest Job First — 실행 시간이 짧은 프로세스 우선. 기아(Starvation) 발생 가능                      |
| **SRTF**                       | Shortest Remaining Time First — SJF의 선점형 버전                                                    |
| **Round Robin (RR)**           | 정해진 시간(Time Quantum)마다 프로세스를 순환 처리. 응답 시간 우수                                   |
| **Priority Scheduling**        | 우선순위가 높은 프로세스 먼저 처리. 기아 문제 → 에이징(Aging)으로 해결                               |
| **Multi-level Queue**          | 여러 개의 준비 큐를 두고 프로세스 유형별로 다른 스케줄링 정책 적용                                   |
| **Multi-level Feedback Queue** | 프로세스가 큐 사이를 이동 가능 — 현대 OS에서 가장 널리 사용                                          |
| **스케줄링 기준 지표**         | CPU 이용률 / 처리량(Throughput) / 응답 시간 / 대기 시간 / 반환 시간                                  |
| **기아 현상 (Starvation)**     | 우선순위가 낮은 프로세스가 계속 CPU를 할당받지 못하는 현상                                           |
| **에이징 (Aging)**             | 대기 시간이 길수록 우선순위를 높여 기아 현상을 방지하는 기법                                         |

## 5. 프로세스 동기화

| 키워드                             | 설명                                                                              |
| ---------------------------------- | --------------------------------------------------------------------------------- |
| **임계 구역 (Critical Section)**   | 공유 자원에 접근하는 코드 영역 — 한 번에 하나의 프로세스만 진입해야 함            |
| **경쟁 조건 (Race Condition)**     | 여러 프로세스가 공유 자원에 동시 접근해 결과가 실행 순서에 따라 달라지는 문제     |
| **임계 구역 해결 조건**            | 상호 배제(Mutual Exclusion) / 진행(Progress) / 한정 대기(Bounded Waiting)         |
| **뮤텍스 (Mutex)**                 | 하나의 스레드만 공유 자원에 접근하도록 잠금(Lock) / 해제(Unlock)하는 동기화 도구  |
| **세마포어 (Semaphore)**           | 정수 카운터를 이용한 동기화 도구 — P(wait) 연산으로 감소, V(signal) 연산으로 증가 |
| **이진 세마포어 vs 계수 세마포어** | 이진(0 또는 1): 뮤텍스와 유사 / 계수: 특정 개수의 자원을 관리                     |
| **모니터 (Monitor)**               | 언어 수준에서 상호 배제를 지원하는 고수준 동기화 구조 (Java의 synchronized)       |
| **스핀락 (Spinlock)**              | 락을 획득할 때까지 CPU를 사용하며 계속 확인하는 바쁜 대기(Busy Waiting) 방식      |
| **생산자-소비자 문제**             | 공유 버퍼를 사이에 두고 생산자와 소비자가 동기화해야 하는 고전적 문제             |
| **철학자 식사 문제**               | 교착 상태와 기아 현상을 설명하는 고전적 동기화 문제                               |
| **readers-writers 문제**           | 여러 reader는 동시 접근 가능하지만 writer는 단독 접근해야 하는 동기화 문제        |

## 6. 교착 상태 (Deadlock)

| 키워드                          | 설명                                                                        |
| ------------------------------- | --------------------------------------------------------------------------- |
| **교착 상태 (Deadlock)**        | 둘 이상의 프로세스가 서로 상대방의 자원을 기다리며 무한 대기하는 상태       |
| **교착 상태 발생 조건**         | 상호 배제 / 점유 대기 / 비선점 / 순환 대기 — 4가지 모두 충족 시 발생        |
| **자원 할당 그래프**            | 프로세스와 자원의 할당 관계를 나타낸 그래프 — 사이클 존재 시 교착 상태 가능 |
| **교착 상태 예방 (Prevention)** | 4가지 조건 중 하나를 원천 차단 — 자원 낭비 발생 가능                        |
| **교착 상태 회피 (Avoidance)**  | 은행원 알고리즘(Banker's Algorithm)으로 안전 상태를 유지하며 자원 할당      |
| **교착 상태 탐지 (Detection)**  | 교착 상태 발생을 허용하고 주기적으로 탐지 후 회복                           |
| **교착 상태 회복 (Recovery)**   | 교착 상태 프로세스를 종료하거나 자원을 강제로 선점해 해결                   |
| **은행원 알고리즘**             | 자원 요청 시 안전 상태인지 미리 시뮬레이션해 교착 상태를 회피하는 알고리즘  |

## 7. 메모리 관리

| 키워드                          | 설명                                                                                          |
| ------------------------------- | --------------------------------------------------------------------------------------------- |
| **연속 메모리 할당**            | 프로세스를 메모리의 연속된 공간에 적재하는 방식                                               |
| **고정 분할 vs 가변 분할**      | 고정: 메모리를 고정 크기로 분할 (내부 단편화) / 가변: 프로세스 크기에 맞게 분할 (외부 단편화) |
| **내부 단편화**                 | 할당된 공간이 실제 사용 크기보다 커서 내부에 낭비 공간이 생기는 현상                          |
| **외부 단편화**                 | 여유 공간이 존재하지만 연속되지 않아 큰 프로세스를 할당하지 못하는 현상                       |
| **메모리 할당 방식**            | 최초 적합(First Fit) / 최적 적합(Best Fit) / 최악 적합(Worst Fit)                             |
| **압축 (Compaction)**           | 분산된 빈 공간을 한쪽으로 모아 큰 연속 공간을 만드는 기법                                     |
| **스와핑 (Swapping)**           | 메모리가 부족할 때 프로세스를 통째로 디스크(스왑 영역)로 내보내는 기법                        |
| **페이징 (Paging)**             | 프로세스와 메모리를 동일한 고정 크기(페이지/프레임)로 나눠 비연속 할당하는 기법               |
| **세그멘테이션 (Segmentation)** | 코드 / 데이터 / 스택 등 논리적 단위로 메모리를 나눠 관리하는 기법                             |
| **페이지 테이블**               | 가상 주소(페이지 번호)를 물리 주소(프레임 번호)로 변환하는 자료구조                           |
| **TLB**                         | 페이지 테이블 조회 속도를 높이기 위해 최근 변환 결과를 캐싱하는 하드웨어                      |

## 8. 가상 메모리

| 키워드                       | 설명                                                                          |
| ---------------------------- | ----------------------------------------------------------------------------- |
| **가상 메모리**              | 물리 메모리보다 큰 주소 공간을 프로세스에 제공하는 기법                       |
| **Demand Paging**            | 실제로 필요한 페이지만 물리 메모리에 적재하는 방식                            |
| **페이지 폴트 (Page Fault)** | 요청한 페이지가 물리 메모리에 없을 때 발생하는 예외                           |
| **페이지 교체 알고리즘**     | OPT / FIFO / LRU / Clock — 교체할 페이지를 선택하는 알고리즘                  |
| **스래싱 (Thrashing)**       | 페이지 폴트가 과도하게 발생해 CPU 효율이 급격히 떨어지는 현상                 |
| **워킹셋 (Working Set)**     | 프로세스가 일정 시간 동안 참조하는 페이지의 집합 — 스래싱 방지에 활용         |
| **Copy-on-Write (COW)**      | fork() 시 메모리를 즉시 복사하지 않고 쓰기가 발생할 때만 복사하는 최적화 기법 |
| **메모리 맵 파일**           | 파일을 가상 메모리에 매핑해 파일 I/O를 메모리 접근처럼 처리하는 기법          |

## 9. 파일 시스템

| 키워드                                | 설명                                                                        |
| ------------------------------------- | --------------------------------------------------------------------------- |
| **파일 시스템**                       | 파일을 저장 장치에 체계적으로 저장하고 관리하는 OS의 구성 요소              |
| **디렉터리 구조**                     | 1단계 / 2단계 / 트리형 / 비순환 그래프형 디렉터리 구조                      |
| **파일 할당 방식**                    | 연속 할당 / 연결 할당 / 색인 할당 (inode)                                   |
| **inode**                             | 파일의 메타데이터(크기, 권한, 블록 위치 등)를 저장하는 자료구조 (Unix 계열) |
| **FAT (File Allocation Table)**       | 파일의 디스크 블록 위치를 테이블로 관리하는 방식 (Windows)                  |
| **저널링 (Journaling)**               | 파일 변경 전 로그를 먼저 기록해 시스템 충돌 시 복구를 가능하게 하는 기법    |
| **마운트 (Mount)**                    | 파일 시스템을 디렉터리 트리에 연결해 접근 가능하게 하는 작업                |
| **파일 디스크립터 (File Descriptor)** | 열린 파일을 식별하는 정수 값 — 프로세스마다 독립적으로 관리                 |
| **버퍼 캐시 / 페이지 캐시**           | 디스크 I/O를 줄이기 위해 자주 접근하는 파일 데이터를 메모리에 캐싱          |
| **하드 링크 vs 소프트 링크**          | 하드 링크: 같은 inode를 가리킴 / 소프트 링크: 경로를 가리키는 단축키        |

## 10. 입출력 & 디스크 관리

| 키워드                     | 설명                                                                              |
| -------------------------- | --------------------------------------------------------------------------------- |
| **I/O 방식**               | Polling / Interrupt-driven / DMA                                                  |
| **블로킹 vs 논블로킹 I/O** | 블로킹: I/O 완료까지 프로세스 대기 / 논블로킹: 완료를 기다리지 않고 즉시 반환     |
| **동기 vs 비동기 I/O**     | 동기: 결과를 직접 기다림 / 비동기: 완료 시 콜백/이벤트로 통보                     |
| **디스크 스케줄링**        | 디스크 헤드 이동을 최소화해 I/O 처리량을 높이는 알고리즘                          |
| **FCFS 디스크**            | 요청 순서대로 처리 — 단순하지만 헤드 이동 거리 최적화 없음                        |
| **SSTF**                   | Shortest Seek Time First — 현재 헤드에서 가장 가까운 요청 먼저 처리               |
| **SCAN (엘리베이터)**      | 헤드가 한 방향으로 이동하며 요청 처리 후 반대 방향으로 이동                       |
| **C-SCAN**                 | SCAN에서 한쪽 방향으로만 처리하고 끝에서 처음으로 복귀                            |
| **RAID**                   | 여러 디스크를 묶어 성능·신뢰성 향상 (0 / 1 / 5 / 6 / 10)                          |
| **버퍼링 (Buffering)**     | I/O 속도 차이를 완충하기 위해 데이터를 임시 저장하는 기법                         |
| **스풀링 (Spooling)**      | 디스크를 중간 저장소로 사용해 여러 작업이 입출력 장치를 공유하는 기법 (프린터 등) |
