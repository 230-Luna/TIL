## 1. 언어 & 컴파일

| 키워드                         | 설명                                                                             |
| ------------------------------ | -------------------------------------------------------------------------------- |
| **고급 언어 vs 저급 언어**     | 사람이 읽기 쉬운 언어(C, Java 등) vs 기계가 직접 실행하는 언어(기계어, 어셈블리) |
| **컴파일 언어**                | 소스 코드 전체를 기계어로 변환 후 실행 (C, C++)                                  |
| **인터프리터 언어**            | 소스 코드를 한 줄씩 해석하며 실행 (Python, JS)                                   |
| **어셈블리어**                 | 기계어를 사람이 읽기 쉽게 기호로 표현한 저급 언어                                |
| **어셈블러 / 컴파일러 / 링커** | 어셈블리→기계어 / 고급→기계어 / 목적 파일을 연결하는 도구                        |

## 2. 수 표현 & 데이터

| 키워드                      | 설명                                                          |
| --------------------------- | ------------------------------------------------------------- |
| **이진법 / 16진법**         | 컴퓨터의 기본 수 표현 체계                                    |
| **2의 보수**                | 음수를 표현하는 방식 — 비트 반전 후 1 더하기                  |
| **부동소수점**              | IEEE 754 — 부호 / 지수부 / 가수부로 실수 표현                 |
| **오버플로우 / 언더플로우** | 표현 범위를 초과하거나 너무 작은 수를 표현할 때 발생하는 오류 |
| **아스키 코드 (ASCII)**     | 영문자·숫자·특수문자를 7비트로 표현하는 문자 인코딩           |
| **유니코드 / UTF-8**        | 전 세계 문자를 표현하는 표준 인코딩 — UTF-8은 가변 길이       |
| **엔디안**                  | Big Endian (MSB 먼저) vs Little Endian (LSB 먼저)             |
| **비트 연산**               | AND / OR / XOR / Shift — 저수준 연산의 기본                   |

## 3. 명령어

| 키워드                 | 설명                                                             |
| ---------------------- | ---------------------------------------------------------------- |
| **연산 코드 (Opcode)** | 명령어에서 수행할 연산의 종류를 나타내는 필드                    |
| **오퍼랜드 (Operand)** | 명령어에서 연산에 사용할 데이터 또는 주소                        |
| **주소 지정 방식**     | 오퍼랜드의 실제 위치를 찾는 방법                                 |
| **즉시 주소 지정**     | 오퍼랜드 필드에 데이터를 직접 명시                               |
| **직접 주소 지정**     | 오퍼랜드 필드에 메모리 주소를 직접 명시                          |
| **간접 주소 지정**     | 오퍼랜드 필드의 주소가 가리키는 곳에 실제 주소 존재              |
| **레지스터 주소 지정** | 오퍼랜드가 레지스터를 가리킴                                     |
| **변위 주소 지정**     | 기준 주소(PC / 베이스 레지스터)에 오퍼랜드를 더해 유효 주소 계산 |
| **스택 주소 지정**     | 스택 포인터가 가리키는 스택 영역 사용                            |
| **ISA**                | Instruction Set Architecture — CPU가 이해할 수 있는 명령어 집합  |
| **CISC vs RISC**       | 복잡한 명령어 집합 vs 단순한 명령어 집합 구조 비교               |

## 4. CPU & 명령어 사이클

| 키워드                         | 설명                                                               |
| ------------------------------ | ------------------------------------------------------------------ |
| **ALU**                        | Arithmetic Logic Unit — 산술·논리 연산을 수행하는 회로             |
| **레지스터**                   | CPU 내부의 초고속 임시 저장 공간 (PC, IR, MAR, MDR, SP 등)         |
| **제어장치**                   | 명령어를 해석하고 각 장치에 제어 신호를 보내는 유닛                |
| **명령어 사이클**              | Fetch → Decode → Execute → Write-back                              |
| **인터럽트**                   | 외부 신호에 의해 CPU 실행 흐름이 변경되는 것                       |
| **인터럽트 서비스 루틴 (ISR)** | 인터럽트 발생 시 실행되는 처리 코드                                |
| **파이프라이닝**               | 명령어를 단계별로 나눠 병렬 처리하는 기법                          |
| **해저드**                     | 파이프라인 충돌 — 데이터 해저드 / 제어 해저드 / 구조적 해저드      |
| **슈퍼스칼라**                 | 여러 파이프라인을 동시에 실행하는 구조                             |
| **비순차 실행**                | Out-of-Order Execution — 의존성 없는 명령어를 먼저 실행            |
| **분기 예측**                  | Branch Prediction — 조건 분기의 결과를 미리 예측해 파이프라인 유지 |

## 5. 메모리 계층

| 키워드               | 설명                                                             |
| -------------------- | ---------------------------------------------------------------- |
| **메모리 계층 구조** | 레지스터 → 캐시 → RAM → Disk 순으로 속도↓ 용량↑                  |
| **캐시 메모리**      | CPU와 RAM 사이의 고속 임시 저장소 (L1 / L2 / L3)                 |
| **지역성**           | 시간적 지역성 (최근 사용) / 공간적 지역성 (인접 주소)            |
| **캐시 교체 정책**   | LRU / FIFO / LFU — 캐시 공간이 부족할 때 교체할 데이터 선택 방식 |
| **캐시 쓰기 정책**   | Write-through (즉시 반영) / Write-back (나중에 반영)             |
| **캐시 매핑 방식**   | 직접 매핑 / 완전 연관 매핑 / 집합 연관 매핑                      |
| **캐시 일관성**      | Cache Coherence — 멀티코어 환경에서 캐시 간 데이터 일치 보장     |
| **TLB**              | Translation Lookaside Buffer — 가상→물리 주소 변환을 캐싱        |

## 6. 가상 메모리

| 키워드                   | 설명                                                               |
| ------------------------ | ------------------------------------------------------------------ |
| **페이징**               | 메모리를 고정 크기 페이지 단위로 분할하여 관리                     |
| **세그멘테이션**         | 논리적 단위(코드/데이터/스택)로 메모리를 분할하여 관리             |
| **페이지 테이블**        | 가상 주소 → 물리 주소 매핑 정보를 담은 자료구조                    |
| **다단계 페이지 테이블** | 페이지 테이블 크기를 줄이기 위한 계층적 구조                       |
| **페이지 폴트**          | 요청한 페이지가 물리 메모리에 없을 때 발생하는 예외                |
| **페이지 교체 알고리즘** | OPT / LRU / FIFO / Clock — 어떤 페이지를 swap out 할지 결정        |
| **스래싱**               | Thrashing — 페이지 폴트가 과도하게 발생해 CPU 효율이 급감하는 현상 |
| **Demand Paging**        | 실제로 필요한 페이지만 물리 메모리에 적재하는 방식                 |

## 7. 저장장치 & 입출력장치

| 키워드            | 설명                                                               |
| ----------------- | ------------------------------------------------------------------ |
| **HDD**           | 플래터 / 섹터 / 트랙 / 실린더 구조의 자기 디스크 저장장치          |
| **SSD**           | 플래시 메모리 기반 저장장치 — 빠르고 소음 없음                     |
| **RAID**          | 여러 디스크를 묶어 성능·신뢰성 향상 (0 / 1 / 5 / 6 / 10)           |
| **장치 컨트롤러** | 입출력 장치와 CPU/메모리 사이를 중재하는 하드웨어                  |
| **장치 드라이버** | OS가 장치 컨트롤러를 제어하기 위한 소프트웨어                      |
| **DMA**           | Direct Memory Access — CPU 개입 없이 I/O 장치가 메모리에 직접 접근 |

| 키워드          | 설명                                             |

## 9. 병렬 처리

| 키워드                       | 설명                                                               |
| ---------------------------- | ------------------------------------------------------------------ |
| **플린 분류**                | SISD / SIMD / MISD / MIMD — 명령어·데이터 흐름에 따른 분류         |
| **멀티코어 vs 멀티프로세서** | 하나의 칩에 여러 코어 vs 여러 개의 물리적 CPU                      |
| **Race Condition**           | 여러 프로세스가 공유 자원에 동시 접근해 결과가 달라지는 문제       |
| **Atomic Operation**         | 중간에 끊기지 않는 원자적 연산 — 동기화의 기본 단위                |
| **메모리 모델**              | 강한 일관성 (Strong Consistency) vs 약한 일관성 (Weak Consistency) |
