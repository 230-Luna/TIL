## 1. 네트워크 기초

| 키워드                                     | 설명                                                                 |
| ------------------------------------------ | -------------------------------------------------------------------- |
| **네트워크**                               | 여러 장치가 통신 매체로 연결되어 데이터를 주고받는 시스템            |
| **프로토콜**                               | 네트워크 통신을 위한 규칙과 약속의 집합                              |
| **대역폭 (Bandwidth)**                     | 단위 시간당 전송 가능한 데이터의 최대량 (bps)                        |
| **처리량 (Throughput)**                    | 실제로 단위 시간에 전달되는 데이터 양                                |
| **지연 시간 (Latency)**                    | 데이터가 출발지에서 목적지까지 도달하는 데 걸리는 시간               |
| **패킷 (Packet)**                          | 네트워크에서 전송되는 데이터의 기본 단위                             |
| **LAN / WAN / MAN**                        | 근거리 통신망 / 광역 통신망 / 도시 통신망                            |
| **유니캐스트 / 멀티캐스트 / 브로드캐스트** | 1:1 통신 / 1:다수 통신 / 1:전체 통신                                 |
| **토폴로지**                               | 네트워크의 물리적/논리적 연결 구조 (스타형 / 링형 / 버스형 / 메시형) |

## 2. OSI 7계층 & TCP/IP 4계층

| 키워드                           | 설명                                                             |
| -------------------------------- | ---------------------------------------------------------------- |
| **OSI 7계층**                    | 네트워크 통신을 7단계로 나눈 표준 모델                           |
| **1계층 물리 (Physical)**        | 비트를 전기 신호로 변환해 전송 — 케이블, 허브                    |
| **2계층 데이터링크 (Data Link)** | 같은 네트워크 내 노드 간 전송 — MAC 주소, 이더넷, 스위치         |
| **3계층 네트워크 (Network)**     | 서로 다른 네트워크 간 경로 결정 — IP, 라우터                     |
| **4계층 전송 (Transport)**       | 종단 간 신뢰성 있는 데이터 전송 — TCP, UDP                       |
| **5계층 세션 (Session)**         | 연결 수립·유지·종료 관리                                         |
| **6계층 표현 (Presentation)**    | 데이터 인코딩 / 암호화 / 압축                                    |
| **7계층 응용 (Application)**     | 사용자와 직접 상호작용 — HTTP, FTP, DNS, SMTP                    |
| **TCP/IP 4계층**                 | 네트워크 액세스 / 인터넷 / 전송 / 응용 계층                      |
| **캡슐화 / 역캡슐화**            | 각 계층을 지나며 헤더를 추가(캡슐화) / 제거(역캡슐화)하는 과정   |
| **PDU**                          | 각 계층의 데이터 단위 — 비트 / 프레임 / 패킷 / 세그먼트 / 메시지 |

## 3. 물리 & 데이터링크 계층

| 키워드                | 설명                                                                 |
| --------------------- | -------------------------------------------------------------------- |
| **MAC 주소**          | 네트워크 인터페이스에 부여된 고유한 48비트 하드웨어 주소             |
| **이더넷 (Ethernet)** | 유선 LAN에서 가장 널리 사용되는 데이터링크 프로토콜                  |
| **허브 (Hub)**        | 들어온 신호를 모든 포트로 브로드캐스트하는 1계층 장비                |
| **스위치 (Switch)**   | MAC 주소를 기반으로 특정 포트에만 프레임을 전달하는 2계층 장비       |
| **ARP**               | Address Resolution Protocol — IP 주소로 MAC 주소를 알아내는 프로토콜 |
| **CSMA/CD**           | 이더넷의 충돌 감지 및 회피 방식                                      |
| **VLAN**              | 물리적 구성과 무관하게 논리적으로 LAN을 분리하는 기술                |

## 4. 네트워크 계층 (IP)

| 키워드                        | 설명                                                                            |
| ----------------------------- | ------------------------------------------------------------------------------- |
| **IP 주소**                   | 네트워크에서 장치를 식별하는 논리적 주소                                        |
| **IPv4**                      | 32비트 주소 체계 — 약 43억 개의 주소                                            |
| **IPv6**                      | 128비트 주소 체계 — IPv4 주소 고갈 문제 해결                                    |
| **서브넷 마스크**             | IP 주소에서 네트워크 부분과 호스트 부분을 구분하는 비트 마스크                  |
| **서브네팅**                  | 하나의 네트워크를 여러 개의 서브넷으로 분할하는 것                              |
| **CIDR**                      | Classless Inter-Domain Routing — 클래스 없이 유연하게 IP 범위를 표기하는 방식   |
| **라우터 (Router)**           | IP 주소 기반으로 최적 경로를 결정해 패킷을 전달하는 3계층 장비                  |
| **라우팅**                    | 패킷의 목적지까지 최적 경로를 결정하는 과정                                     |
| **라우팅 테이블**             | 목적지 IP에 따른 다음 홉(Next Hop) 정보를 담은 테이블                           |
| **정적 라우팅 / 동적 라우팅** | 관리자가 직접 설정 / 라우팅 프로토콜(RIP, OSPF, BGP)이 자동으로 경로 설정       |
| **NAT**                       | Network Address Translation — 사설 IP를 공인 IP로 변환해 인터넷에 접속하는 기술 |
| **ICMP**                      | 네트워크 상태 진단용 프로토콜 — `ping`, `traceroute` 등에 사용                  |
| **공인 IP / 사설 IP**         | 인터넷에서 사용하는 전 세계 유일한 IP / LAN 내부에서만 사용하는 IP              |
| **게이트웨이**                | 서로 다른 네트워크를 연결하는 장치 — 일반적으로 라우터가 담당                   |

## 5. 전송 계층 (TCP & UDP)

| 키워드                             | 설명                                                                     |
| ---------------------------------- | ------------------------------------------------------------------------ |
| **포트 (Port)**                    | 하나의 IP에서 프로세스를 구분하는 번호 (0~65535)                         |
| **TCP**                            | 연결 지향, 신뢰성 있는 전송 프로토콜 — 순서 보장, 오류 감지, 흐름 제어   |
| **UDP**                            | 비연결 지향, 빠른 전송 프로토콜 — 신뢰성 없지만 오버헤드 적음            |
| **3-way handshake**                | TCP 연결 수립 과정 — SYN → SYN-ACK → ACK                                 |
| **4-way handshake**                | TCP 연결 종료 과정 — FIN → ACK → FIN → ACK                               |
| **흐름 제어 (Flow Control)**       | 수신자의 처리 속도에 맞게 송신 속도를 조절하는 것 — 슬라이딩 윈도우 사용 |
| **혼잡 제어 (Congestion Control)** | 네트워크 혼잡을 감지하고 송신 속도를 줄이는 것 — Slow Start, AIMD        |
| **오류 제어**                      | 손실/손상된 패킷을 감지하고 재전송하는 메커니즘 — ARQ                    |
| **슬라이딩 윈도우**                | ACK를 기다리지 않고 연속해서 여러 패킷을 전송하는 흐름 제어 기법         |
| **RTT (Round Trip Time)**          | 패킷을 보내고 ACK를 받을 때까지 걸리는 시간                              |
| **TIME_WAIT**                      | TCP 연결 종료 후 일정 시간 동안 포트를 열어두는 상태 (지연 패킷 처리)    |
| **소켓 (Socket)**                  | 네트워크 통신의 끝점 — IP 주소 + 포트 번호로 식별                        |

## 6. 응용 계층

| 키워드                 | 설명                                                      |
| ---------------------- | --------------------------------------------------------- |
| **HTTP**               | HyperText Transfer Protocol — 웹 통신의 기본 프로토콜     |
| **HTTPS**              | HTTP + TLS/SSL 암호화 — 보안이 강화된 HTTP                |
| **FTP**                | 파일 전송 프로토콜                                        |
| **SMTP / POP3 / IMAP** | 이메일 송신 / 수신 프로토콜                               |
| **DNS**                | 도메인 이름을 IP 주소로 변환하는 시스템                   |
| **DHCP**               | 네트워크 장치에 IP 주소를 자동으로 할당하는 프로토콜      |
| **SSH**                | 원격 서버에 안전하게 접속하기 위한 암호화 프로토콜        |
| **WebSocket**          | 클라이언트-서버 간 양방향 실시간 통신을 지원하는 프로토콜 |

## 7. HTTP & HTTPS

| 키워드                   | 설명                                                                       |
| ------------------------ | -------------------------------------------------------------------------- |
| **HTTP 메서드**          | GET / POST / PUT / PATCH / DELETE / HEAD / OPTIONS                         |
| **HTTP 상태 코드**       | 1xx 정보 / 2xx 성공 / 3xx 리다이렉션 / 4xx 클라이언트 오류 / 5xx 서버 오류 |
| **HTTP/1.0**             | 요청마다 TCP 연결을 새로 맺는 방식                                         |
| **HTTP/1.1**             | Keep-Alive(지속 연결), 파이프라이닝 도입                                   |
| **HTTP/2**               | 멀티플렉싱, 헤더 압축(HPACK), 서버 푸시 지원                               |
| **HTTP/3**               | TCP 대신 QUIC(UDP 기반) 프로토콜 사용 — 연결 지연 최소화                   |
| **쿠키 (Cookie)**        | 서버가 클라이언트에 저장하는 작은 데이터 — 상태 유지에 사용                |
| **세션 (Session)**       | 서버 측에서 클라이언트 상태를 저장하는 방식                                |
| **JWT (JSON Web Token)** | 서버가 발급하는 자가 서명된 토큰 — 무상태(Stateless) 인증에 사용           |
| **REST API**             | HTTP 메서드와 URI를 활용한 웹 API 설계 아키텍처 스타일                     |
| **CORS**                 | Cross-Origin Resource Sharing — 다른 출처의 리소스 요청을 허용하는 정책    |
| **TLS / SSL**            | 데이터 암호화, 서버 인증, 무결성 보장을 위한 보안 프로토콜                 |
| **무상태 (Stateless)**   | HTTP는 각 요청이 독립적으로 처리되며 이전 요청 상태를 저장하지 않음        |

## 8. DNS

| 키워드                        | 설명                                                                              |
| ----------------------------- | --------------------------------------------------------------------------------- |
| **DNS**                       | 도메인 이름(www.google.com)을 IP 주소로 변환하는 분산 데이터베이스 시스템         |
| **DNS 조회 과정**             | 브라우저 캐시 → OS 캐시 → 로컬 DNS 서버 → 루트 → TLD → 권한 있는 DNS 서버         |
| **재귀적 질의 / 반복적 질의** | 로컬 DNS 서버가 대신 조회해주는 방식 / 클라이언트가 직접 단계적으로 조회하는 방식 |
| **DNS 레코드 타입**           | A (IPv4) / AAAA (IPv6) / CNAME (별칭) / MX (메일) / NS (네임서버) / TXT           |
| **TTL (Time To Live)**        | DNS 캐시가 유지되는 시간                                                          |
| **DNS 캐싱**                  | 조회 결과를 일정 시간 저장해 반복 조회를 줄이는 것                                |

## 9. 보안

| 키워드                   | 설명                                                                    |
| ------------------------ | ----------------------------------------------------------------------- |
| **대칭키 암호화**        | 암호화와 복호화에 같은 키를 사용 — 빠르지만 키 교환 문제 존재 (AES)     |
| **비대칭키 암호화**      | 공개키로 암호화, 개인키로 복호화 — 느리지만 키 교환 안전 (RSA)          |
| **공개키 / 개인키**      | 누구에게나 공개하는 키 / 소유자만 보관하는 비밀 키                      |
| **SSL/TLS 핸드셰이크**   | HTTPS 연결 수립 시 인증서 교환 및 세션 키를 협의하는 과정               |
| **인증서 (Certificate)** | CA(인증 기관)가 서버의 공개키를 보증하는 디지털 문서                    |
| **해시 (Hash)**          | 임의의 데이터를 고정 길이 값으로 변환하는 단방향 함수 (SHA-256 등)      |
| **방화벽 (Firewall)**    | 네트워크 트래픽을 필터링해 허가된 통신만 허용하는 보안 장비             |
| **XSS**                  | Cross-Site Scripting — 악성 스크립트를 웹 페이지에 삽입하는 공격        |
| **CSRF**                 | Cross-Site Request Forgery — 사용자의 인증 정보를 이용한 위조 요청 공격 |
| **SQL Injection**        | 악의적인 SQL 구문을 입력해 데이터베이스를 조작하는 공격                 |
| **DDoS**                 | 분산 서비스 거부 공격 — 대량의 트래픽으로 서버를 마비시키는 공격        |

## 10. 기타 네트워크 개념

| 키워드                  | 설명                                                                            |
| ----------------------- | ------------------------------------------------------------------------------- |
| **프록시 서버**         | 클라이언트와 서버 사이에서 요청을 중계하는 서버 — 캐싱, 보안, 필터링 역할       |
| **리버스 프록시**       | 클라이언트 요청을 받아 내부 서버로 전달하는 서버 — 로드밸런서, Nginx 등         |
| **로드 밸런싱**         | 여러 서버에 트래픽을 균등하게 분산시키는 기술                                   |
| **CDN**                 | Content Delivery Network — 전 세계 여러 서버에 콘텐츠를 분산 저장해 빠르게 제공 |
| **VPN**                 | 공용 네트워크 위에서 암호화된 사설 터널을 생성하는 기술                         |
| **방화벽**              | 허가된 트래픽만 통과시키는 네트워크 보안 장비                                   |
| **P2P**                 | 서버 없이 노드들이 직접 통신하는 분산 네트워크 구조                             |
| **포트 포워딩**         | 특정 포트로 들어오는 트래픽을 내부 서버의 특정 포트로 전달하는 설정             |
| **소켓 프로그래밍**     | 소켓 API를 이용해 네트워크 통신 애플리케이션을 개발하는 것                      |
| **멱등성 (Idempotent)** | 같은 요청을 여러 번 보내도 결과가 동일한 성질 — GET, PUT, DELETE는 멱등         |
